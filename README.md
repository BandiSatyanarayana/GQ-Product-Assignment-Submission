<h1 align="center">Product Teardown: Google Pay – UPI Payment Journey</h1>
<h3>Product Overview</h3>
<p>This project analyzes Google Pay, a widely used fintech application in India that enables fast digital payments using UPI. The teardown focuses on the Send Money user journey, evaluating how users transfer money to contacts and merchants through the app.</p>
<h3>Core User Journey</h3>
<p>The payment process follows a simple flow:
Open App → Select Contact / UPI ID → Enter Amount → Enter UPI PIN → Payment Confirmation.
This journey is designed for speed and simplicity, allowing users to complete transactions within a few seconds.</p>
<h3>What the Product Does Well</h3>
<p>Google Pay provides a clean interface, quick payment processing, and strong trust signals like confirmation screens and notifications. QR-based payments and easy contact selection also make everyday transactions smooth for users.</p>
<h3>Friction Points & Improvements</h3>
<p>Some issues include transaction failures, risk of sending money to the wrong recipient, limited spending insights, and slow refund updates. Improvements such as recipient confirmation screens, real-time transaction tracking, refund prediction, and AI spending insights can improve user experience and reliability.</p>
<h3>AI Opportunities & Roadmap</h3>
<p>AI can enhance the payment journey through fraud detection alerts, AI payment assistants, and automated financial insights. The roadmap prioritizes reliability improvements first, safety features next, and AI-driven engagement features later.</p>
